diff

现代框架的一个原理 virtual dom
算出 virtual dom ，对比两个的差异，再进行绘制
所以数据发生一点变化 就会重新算一遍数据结构，得到新的dom结构
效率O(n) 只会同级比较 ，虽然不是最优，但效率更高

而jQuery则是，知道变化的dom在哪里，直接操作，没有计算diff的过程，性能更好。但是代码可维护性差，都是因为人肉知道diff 而修改。
之所以现代新流行的框架会 用这种数据更改 ，对比dom差异，重新绘制的方式，因为 是以数据为核心

两行连着修改，也只会调用一次nexttick

性能优化
key用于计算diff的时候更快
解决 li v-for 情况 如果UNshift的话 一样的数据就不会改了，不然会全改
push的话可以不用key，因为是顺序对比

有关组件化开发的意义

一个重要的事情值得注意，关注点分离不等于文件类型分离。

在现代 UI 开发中，我们已经发现相比于把代码库分离成三个大的层次并将其相互交织起来，

把它们划分为松散耦合的组件再将其组合起来更合理一些。

在一个组件里，其模板、逻辑和样式是内部耦合的，并且把他们搭配在一起实际上使得组件更加内聚且更可维护。

路由

路由就是根据网址的不同，返回不同的内容给用户

前端路由

用Vue写的是单页应用。页面进行跳转的时候，不会发生html的请求

通过JS感知到页面的变化，在将当前页面清除，将下一个页面挂在上来

缺点是 首屏展示慢 SEO差

优点是一旦加载好，之后页面的跳转，更快，页面模板部分不会变，数据通过AJAX获取，会给人没有刷新的感觉

嵌套路由 一个路由中嵌套另一个路由

单文件入口，将网址分享给其他人的时候，不管是哪个页面，请求的都应该是入口文件。

URL分享给其它人（直接在某个地址刷新浏览器）， 服务器返回正确的入口页面 应用本身部署在哪个地址下，让前端配置base地址，即让前端路由工具区分哪部分是后端路由，哪部分是前端路由。

相对路径，页面里的资源最好不使用相对路径，最好直接相对于网站根目录

router-link 相当于a标签 如果不想让他做a标签 就在里面加 tag=”xx”
