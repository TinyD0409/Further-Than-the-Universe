从零开始搭建自己的服务器

发表于 2018-10-03 | 分类于 Note
  1. 首先 买一个服务器 如搬瓦工上。获得ip 123.456.67.890 端口号 25000

先关闭服务器，为其下载ubuntu系统

登录到你的服务器上 $ssh root@123.456.67.890 -p 25000

会提示你输入初始密码

进入第一件事修改密码 pwd

创建公私钥，以便不输入密码直接登录

$ssh keygen.exe(git自带的命令)

$mdir .ssh

$ssh-copy-id root@123.456.67.890 -p 25000

在本地储存公私钥的文件夹创建/有就不用创建了 config

host 123.456.67.890 Port 25000 这样登录时候就不用写-p25000了

修改之前申请的域名的DNS，将其指向这个ip

在Linux里下载是叫apt

apt install nodejs（node最新版本的下载参照官网

apt install curl/wget （下载工具

apt install -zsh （主题修改 显示的信息会多一点,更多配置 可以搜索oh my zsh

ZSH_THEME = ‘ys’

$scp (-r递归式上传，上传文件夹时用)-p 25000 ./xxx.txt root@123.456.67.890 :/root 上传文件到root文件夹

sftp://

FileZila与远程系统共享 文件

winSCP 在本地映射成一个本地磁盘 这两种方法都可以直接将文件上传到远程服务器

mcedit 一个在Linux运行的图形化编辑器

如何让服务器24小时后台开启

1 tmux 可以同时运行几个服务器，可以实现远程结队编程

apt install tmux

tmux中组合命令键为 Ctrl+B

Ctrl+B+ %水平拆分成2个

tmux Ctrl+B+D 退出并保持tmux窗口

tmux a 重新挂载（显示出运行着的tmux窗口）

2 pm2 优势在于会自动重启

pm2 start hs.js 打开并运行hs

pm2 monit 显示全部进程的日志

pm2 log 0 显示id为0的进程日志

pm2 l 显示之前的列表

pm2 restart 0 重启id为0的进程

pm2 stop 0 停止id为0的进程

如何使自己的域名以https方式打开

下载$curl https://get.acme.sh | sh

安装$ apt install socat

获得证书$acme.sh --issue -d 49.tinyd.ooo --standalone

记住键值对存储的路径。http-server.js时 require的是https模块

http请求的默认端口80

https请求的默认端口443

https模块需要有key和cert的参数

可以使用 hsts 让页面自动跳转到https

也可以使用 location.protocal = ‘https:’

Your cert is in /root/.acme.sh/49.tinyd.ooo/49.tinyd.ooo.cer

Your cert key is in /root/.acme.sh/49.tinyd.ooo/49.tinyd.ooo.key

The intermediate CA cert is in /root/.acme.sh/49.tinyd.ooo/ca.cer

And the full chain certs is there: /root/.acme.sh/49.tinyd.ooo/fullchain.cer
