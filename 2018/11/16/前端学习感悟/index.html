前端学习感悟

发表于 2018-11-16 | 分类于 Dairy
  • 实践是检验真理的唯一标准。多写，多想，多练，多试，多总结，多动脑。
  • 有些人就是创造历史造福人类的。但是世界是由大部分更普通的人一砖一瓦搭建的。所以创造不了历史，写不出帅气代码也不要太难过，即使是只能做到及格也并没什么可耻。认识差距，接受差距，客观看待差距，做到自己力所能及的事情，既要仰望星空，更要脚踏实地。
  • 70%CSS的问题是选择器的问题，剩下的10%是标点符号问题，再10%可能各种各样的细节问题，最后只有10%是思路完全的错误。当然有时候没法写选择器可能也是完全的思路问题……
    现在想想那个改程序员朋友文档半角标点符号为全角符号，导致程序员自杀的段子……真是残忍的段子啊…… ​​​​
  • 不要怕麻烦，不要怕重复。
    正是这些繁琐的积累，才一步步的铺就了成神的道路。 ​​​​
  • 解决问题的能力。
    如何把猪变成锅包肉。看似遥远不可及的问题，也被先驱们解决了。只要方向对，一步步来问题总会被解决的。所以要明确方向，你要把一条鱼变成一盘锅包肉，神也是完不成的……

  • 根据二八法则，大佬没那么多，不用怕，所以不要觉得只有自己菜

  • 成神之路和吃包子一样。假如吃到第五个包子才会饱。但你直接吃第五个包子并不会饱。前面并没什么用的铺垫也是前进道路必须的基石……
    所以为什么不直接学简单好用的方法啊啊啊啊为什么要学一堆超难的方法啊啊啊啊

  • 先搬好砖。。。
  • 有些困难你做了才能遇到，没做永远不会遇到。说不上哪种情况更好，但是就像玩游戏一样，肯定是遇到游戏里每个隐藏关卡每一个彩蛋更有趣一点。
  • 有些时候简洁的代码实现不了，那就在他们外面包装一层，再不行包两层。人也是一样，包过装的是不是更好看？！
  • 跪着哭着骂着写也要写下去。写着写着就写出来了。不要觉得自己方法麻烦，慢慢优化嘛。
  • 增强逆向思维。加法不行用减法，乘法不行用除法。数都是算出来的，怎么算都是一样的…… ​​​​
  • 牛顿真厉害。数学真美妙。
  • 一个变量不行就再加一个变量，一个函数不行就再套个函数。做题不也是需要辅助线的？！
    有时候没必要那么拼，绕点路，为生活画条辅助线
  • 量变会引起质变。但是丑小鸭也还是因为本来就是鹅才变成的白天鹅。
    所以因为钱的积攒才会变钱多，破烂的量的变多，并不会变得有钱。
    题要做，基础要打，n手抓n手都要硬。就像课本上那个表示本体 手上抓一堆变量的章鱼一样🐙。
  • 路是人走出来的。代码也是人写出来的。
    世上本没有大佬，写的多了，也就变成了大佬😔(也没有秃子，想的多了，就变成了秃子。。。也没谁更强，变秃了，才变强的。。。。)
    写着写着就有思路了。像高中做题一样，至少写个解上去不是？！
  • 自己留下的坑，早晚也还是会坑自己的。早晚都要填。😔
  • 表面的游刃有余 其实背后都付出了同样的辛苦。看上去快速迅捷 一键完成，其实该走的路一步没少，该吃的苦都是要吃的，该踩的坑都是要踩的。不要老想着走捷径，搬好砖。
  • 遇到困难不要退缩。你觉得难，别人也觉得难。都是难过来的
  • if是最好的语句也是最差的语句。总想着“如果”，那注定不会前进。而又没有什么问题不能如果一下╮(╯▽╰)╭ 如果能重来，我要选李白！！
  • 所谓hard，只不过比easy、medium麻烦事多点，hard也是由无数的easy组合而成的啊，没有什么东西出生就是hard的。台风都是不断加强才会有名字的！~ 当然，不管什么最后不都还是要accept╮(╯▽╰)╭
  • 多线程什么的不存在的……浏览器都做不到，所以我做不到是正常的╮(╯▽╰)╭所以单线程也没什么不好。
  • 不管什么问题最后都会归结到哲学问题……你们这些哲学家真的是吃个饭都不让人安生。 ​​​​
  • 别人的经验不一定完全适用，不要过分迷信。但很可能它们会让你少走弯路。
    想想文章这种千古一大抄的东西，代码也是一样，毕竟核心就那么几条语句，重点在让他们好使。想想当年尽管写得出漂亮的句子，作文依然得不到高分，问题就在中心思想。糊弄出来，再想办法解决问题就好了，毕竟比没有问题要好……╮(╯▽╰)╭
  • 错误的结果也是结果。
  • 我们写的网页，这些html、CSS、js代码，只是实现浏览一个页面最微小的部分。从一个网页到一个网站，不是各子网页的跳转，而是需要传输的数据，这些本质就是01电流的数据，通过各种协议，去编码，构成人机间的桥梁，传到服务器端。服务器和浏览器的互相配合传输通信，才展现出我们正常网络上所看到的页面，才得到我们想知道的内容。
    用户能看到的只是一个呈现出的最表面的岁月静好的样子。而为了给用户这个结果，是需要无数微小的部分各自完成的。计算机不是人，但它的工作方式是人类设定的，可以看到人类社会的影子。吃饭的时候会感恩于农民的劳动，自然的馈赠，运输、厨师等等的过程，点去一个网页这样一个我们每天都做了无数次的行为，也有劳于浏览器、服务器和无数协议等的正确运行。
    作为前端工程师可能能做到的并不是很多，但一想到自己在 这个人们每天重复无数次的活动里 献出着一份力量，也会有种参与感，好像一个巨大装置的一个动力环节。引导无数的01并被无数01引导着，去往正确的地方。自己做的事情也很伟大啊。渺小而伟大。
  • 安全问题比所有问题都更难一些。
  • 我也可以说，那天我好烦躁，什么都不想干，不想玩手机不想看新番，只想写代码，写了一天把xxx 写出来了
  • 现代的大部分应用都是以数据为核心，所以可以因为数据的更改而不惜巨大运算而重新算出dom结构。jQ虽然是一代的英雄，但毕竟时代已经不一样了……唉，一朝天子一朝臣的赶脚……
  • 高内聚，低耦合。多聚合，少继承。 => 可复用和不依赖性/牵一发而动全身什么的要不得啊……
  • 在编程的生涯中，不要忘了小黄鸭永远与你同在……
  • 退一步海阔天空，返回商议般呀~
