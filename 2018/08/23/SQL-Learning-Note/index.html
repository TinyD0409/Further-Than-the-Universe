发表于 2018-08-23 | 分类于 Note

数据库

R DB M S 关系型数据管理系统

Struckture Query Language 结构化数据查询语言

一般语言（客户端）与数据库（服务端）连接通过 自己/第三方的协议

通过SDK与数据库互连 从数据库通过SQL语言查询到数据到客户端，或将数据写回到数据库

通常项目的数据库服务器（磁盘密集型）和网络服务器（网络密集型）不在一个服务器运行

数据库主要的功能 增删改查

常用命令

.help

.exit

.save ./xxx.splite3

.mode column

.header on

create table xxx(列名 int primarykey autoincrease//自增，列名string，列名) 可以设置类型偏好，会尽力满足

有个隐藏属性 rowid

alter table 表名 add column 列名(integer) 添加一列

select * (distinct列名as xx,列名) from 表名 as是将选出的列 换一个列名 distinct是去重

select * from 表名 where 列名 in (‘xx’,’xx’) 在这个表中查找 那一列中 xx和xx的数据

where not 列名 = ‘xx’ and not 列名 = ‘xx’ 某列中 即不包含xx也不包含xx

insert into 表名(列名，列名) values (xx,xx,xx) （null表示那列的值为空

SELECT Orders.OrderID, Customers.CustomerName, Orders.OrderDate FROM Orders INNER JOIN Customers ON Orders.CustomerID=Customers.CustomerID;

有 inner join left join right join 两个表连接， （左表连就是以左表为基准，右边有就有，没有就空着

select * from users as a join users as b 自己连接自己

create view postShow as select * from posts join users on posts.user.id = user.id;

select * from postShow view类似把查询语句保存下来，之后再想要这个结果表的时候不用再写复杂的语句了

这个表还可以继续查询

select last_row_id

order by xx desc limit 1(只要查询结果的1个)
